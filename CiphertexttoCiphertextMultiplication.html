D-2.7 Ciphertext-to-Ciphertext Multiplication

- Reference 1: Introduction to the BFV encryption scheme [12]

- Reference 2: Somewhat Partially Fully Homomorphic Encryption [13]

Given two ciphertexts 𝖱𝖫𝖶𝖤S,σ(ΔM⟨1⟩) and 𝖱𝖫𝖶𝖤S,σ(ΔM⟨2⟩), the goal of ciphertext-to-ciphertext multiplication is to derive a new ciphertext whose decryption is ΔM⟨1⟩M⟨2⟩. Ciphertext-to-ciphertext multiplication is more complex than ciphertext-to-plaintext multiplication. It comprises four steps: (1) ModRaise; (2) polynomial multiplication; (3) relinearization; and (4) rescaling.

For better understanding, we will explain BFV’s ciphertext-to-ciphertext multiplication based on the alternate version of RLWE (Theorem B-4.4 in §B-4.4), where the sign of the AS term is flipped in the encryption and decryption formulas.

D-2.7.1 ModRaise

We learned from Summary D-2.3 (in §D-2.3) that a BFV ciphertext whose ciphertext modulus is q has the (decryption) relation: ΔM + E = A ⋅S + B −K ⋅q, where K ⋅q stands for modulo reduction by q. ModRaise is a process of forcibly raising the modulus of a ciphertext from q →Q, where q ≪Q. Suppose we modify the modulus of ciphertext (A,B) from q to Q, where Q = q ⋅Δ (remember Δ = ⌊q t⌋). Then, the decryption of the mod-raised ciphertext will output A ⋅S + B mod Q. However, since each polynomial coefficient of A and B is less than q and each polynomial coefficient of S is either {−1,0,1}, the resulting polynomial of A ⋅S + B is guaranteed to have each coefficient strictly less than Q even without modulo reduction by Q– this is because (q −1) ⋅n + (q −1) < Q, where (q −1) ⋅n is the maximum possible coefficient of A ⋅S and (q −1) is the maximum possible coefficient of B. And as mentioned before, we know the relation: A ⋅S + B = ΔM + E + Kq. Therefore, the decryption of the mod-raised ciphertext (A,B) mod Q is as follows:

ΔM + E + Kq mod Q = ΔM + E + Kq ▹ since ΔM + E + Kq < Q

The first step of BFV’s ciphertext-to-ciphertext multiplication is to mod-raise the two input ciphertexts (A⟨1⟩,B⟨1⟩) mod q and (A⟨2⟩,B⟨2⟩) mod q from q →Q (where Q = q ⋅Δ) as follows:

(A⟨1⟩,B⟨1⟩) mod Q

(A⟨2⟩,B⟨2⟩) mod Q

After ModRaise, the decryption of these two ciphertexts would be the following:

A⟨1⟩⋅S + B⟨1⟩ = ΔM⟨1⟩+ E⟨1⟩+ K1q < Q

A⟨2⟩⋅S + B⟨2⟩ = ΔM⟨2⟩+ E⟨2⟩+ K2q < Q

Therefore, the mod-raised ciphertexts have the following form:

𝖱𝖫𝖶𝖤S,σ(ΔM⟨1⟩+ K1q) = (A⟨1⟩,B⟨1⟩) mod Q

𝖱𝖫𝖶𝖤S,σ(ΔM⟨2⟩+ K2q) = (A⟨2⟩,B⟨2⟩) mod Q

D-2.7.2 Polynomial Multiplication

Our next goal is to derive a new ciphertext which encrypts (ΔM⟨1⟩+ E⟨1⟩+ K1q) ⋅(ΔM⟨2⟩+ E⟨2⟩+ K2q).

First, we can derive the following relation:

(ΔM⟨1⟩+ E⟨1⟩+ K1q) ⋅(ΔM⟨2⟩+ E⟨2⟩+ K2q)

= (A⟨1⟩⋅S + B⟨1⟩) ⋅(A⟨2⟩⋅S + B⟨2⟩)

= B⟨1⟩B⟨2⟩ D0 + (B⟨2⟩A⟨1⟩+ B⟨1⟩A⟨2⟩) D1 ⋅S + (A⟨1⟩⋅A⟨2⟩) D2 ⋅S ⋅S

= D0 + D1 ⋅S + D2 ⋅S2

Meanwhile, we also have the following relations:

𝖱𝖫𝖶𝖤S,σ−1(ΔM⟨1⟩+ K1q) = ΔM⟨1⟩+ E⟨1⟩+ K1q

𝖱𝖫𝖶𝖤S,σ−1(ΔM⟨2⟩+ K2q) = ΔM⟨2⟩+ E⟨2⟩+ K2q

Combining all these, we reach the following relation:

𝖱𝖫𝖶𝖤S,σ−1(ΔM⟨1⟩+ K1q) ⋅𝖱𝖫𝖶𝖤S,σ−1(ΔM⟨2⟩+ K2q) = D0 + D1 ⋅S + D2 ⋅S2

Notice that D0,D1, and D2 are known values as ciphertext components, whereas S is only known to the private key owner. Therefore, we can view D0 + D1 ⋅S + D2 ⋅S2 as a decryption formula such that given the ciphertext components D0,D1,D2 and the private key S, one can derive (ΔM⟨1⟩+ E⟨1⟩+ K1q) ⋅(ΔM⟨2⟩+ E⟨2⟩+ K2q). In other words, we can let (D0,D1,D2) be a new form of ciphertext which can be decrypted by S into (ΔM⟨1⟩+ E⟨1⟩+ K1q) ⋅(ΔM⟨2⟩+ E⟨2⟩+ K2q).

However, (D0,D1,D2) is not in the RLWE ciphertext format, because it has 3 components instead of 2. Having 3 ciphertext components is computationally inefficient, as its decryption involves a square root of S (i.e., S2). Over consequent ciphertext-to-ciphertext multiplications, this S term will double its exponents as S4,S8,⋯ as well as the number of ciphertext components, which would exponentially increase the computational overhead of decryption. Therefore, we want to convert the intermediate ciphertext format (D0,D1,D2) into a regular BFV ciphertext format that has two polynomials as ciphertext components. This conversion process is called a relinearization process (which will be explained in the next subsection).

D-2.7.3 Relinearization

Relinearization is a process of converting the polynomial triplet (D0,D1,D2) ∈R⟨n,Q⟩3 into two RLWE ciphertexts 𝖼𝗍α and 𝖼𝗍β which hold the relation: D0 + D1S + D2S2 = 𝖱𝖫𝖶𝖤S,σ−1(𝖼𝗍α + 𝖼𝗍β).

In the formula D0 + D1S + D2S2, we can re-write D0 + D1S as a synthetic RLWE ciphertext 𝖼𝗍α = (D1,D0), which can be decrypted by S into D1S + D0. Similarly, our next task is to derive a synthetic RLWE ciphertext 𝖼𝗍β whose decryption is D2 ⋅S2 (i.e., 𝖱𝖫𝖶𝖤S,σ−1(𝖼𝗍β) = D2 ⋅S2).

A naive way of creating a ciphertext that encrypts D2 ⋅S2 is as follows: we encrypt S2 into an RLWE ciphertext as 𝖱𝖫𝖶𝖤S,σ(S2) = (A⟨s⟩,B⟨s⟩) such that A⟨s⟩⋅S + B⟨s⟩ = S2 + E⟨s⟩mod Q (where the ciphertext modulus is Q and the plaintext scaling factor Δ = 1). Then, we perform a ciphertext-to-plaintext multiplication (§C-3) with D2, treating D2 as a plaintext polynomial in modulo Q. However, this approach does not work in practice, because computing D2 ⋅𝖱𝖫𝖶𝖤S,σ(S2) generates a huge noise as follows:

D2 ⋅(A⟨s⟩,B⟨s⟩) = (D2 ⋅A⟨s⟩,D2 ⋅B⟨s⟩)

, whose decryption is:

D2 ⋅A⟨s⟩⋅S + D2 ⋅B⟨s⟩ = D2 ⋅S2 + D2 ⋅E⟨s⟩(modQ)

. In the above decrypted expression D2S2 + D2E⟨s⟩mod Q, the term D2S2 is okay to be reduced modulo Q, because this term is originally allowed to be reduced modulo Q in the final decryption formula D0 + D1S + D2S2 mod Q as well. However, the problematic term is the noise D2 ⋅E⟨s⟩, because its coefficients can be any value in [0,Q −1] (since each coefficient of polynomial D2 = A⟨1⟩A⟨2⟩ can be any value in [0,Q −1]). Such a huge noise is not allowed for correct final decryption.

To avoid this noise issue, an improved solution is to express the RLWE ciphertext that encrypts D2S2 as additions of multiple RLWE ciphertexts with small noises by using the gadget decomposition technique (§A-6.4). For this, we use an RLev ciphertext (§B-5) that encrypts S2. Suppose our gadget vector is g→ = (Q β , Q β2, Q β3,⋯,Q βl). Remember that our goal is to find 𝖼𝗍β = 𝖱𝖫𝖶𝖤S,σ(S2 ⋅D2) given known D2, unknown S, and known 𝖱𝖫𝖾𝗏S,σβ,l(S2) = {𝖱𝖫𝖶𝖤S,σ ( Q βi ⋅S)}i=1l. Then, we can derive 𝖼𝗍β as follows:

𝖼𝗍β = 𝖱𝖫𝖶𝖤S,σ(S2 ⋅D2)

= 𝖱𝖫𝖶𝖤S,σ (S2 ⋅ (D2,1Q β + D2,2 Q β2 + ⋯D2,l Q βl)) ▹ by decomposing D2

= 𝖱𝖫𝖶𝖤S,σ (S2 ⋅D2,1 ⋅Q β ) + 𝖱𝖫𝖶𝖤S,σ (S2 ⋅D2,2 ⋅Q β2) + ⋯ + 𝖱𝖫𝖶𝖤S,σ (S2 ⋅D2,l ⋅Q βl)

= D2,1 ⋅𝖱𝖫𝖶𝖤S,σ (S2 ⋅Q β ) + D2,2 ⋅𝖱𝖫𝖶𝖤S,σ (S2 ⋅Q β2) + ⋯ + D2,l ⋅𝖱𝖫𝖶𝖤S,σ (S2 ⋅Q βl) ▹ where each RLWE ciphertext is an encryption of S2Q β ,S2 Q β2,⋯,S2 Q βl as plaintext with the plaintext scaling factor Δ = 1

= ⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(D2), 𝖱𝖫𝖾𝗏S,σβ,l(S2)⟩ ▹ inner product of Decomp and RLev treating them as vectors

If we decrypt the above, we get the following:

𝖱𝖫𝖶𝖤S,σ−1(𝖼𝗍β = ⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(D2), 𝖱𝖫𝖾𝗏S,σβ,l(S2)⟩) ▹ the scaling factors of 𝖱𝖫𝖾𝗏S,σβ,l(S2) are all 1

= D2,1 ⋅ (E1′+ S2Q β ) + D2,2 ⋅ (E2′+ S2 Q β2) + ⋯ + D2,l ⋅ (El′+ S2 Q βl) # where each Ei′ is a noise embedded in 𝖱𝖫𝖶𝖤S,σ (S2 ⋅Q βi)

= ∑ i=1l(Ei′⋅D2,i) + S2 ⋅ (D2,1Q β + D2,2 Q β2 + ⋯ + D2,l Q βl)

= ∑ i=1l𝜖i + D2 ⋅S2 # where each 𝜖i = Ei′⋅D2,i

≈D2 ⋅S2 # ∑ i=1l𝜖i ≪D2 ⋅E″, where E″ is the noise that could’ve been embedded in 𝖱𝖫𝖶𝖤S,σ(S2)

Therefore, we get the following comprehensive relation:

𝖱𝖫𝖶𝖤S,σ−1(ΔM⟨1⟩+ K1q) ⋅𝖱𝖫𝖶𝖤S,σ−1(ΔM⟨2⟩+ K2q) mod Q

= (ΔM⟨1⟩+ E⟨1⟩+ K1q) ⋅(ΔM⟨2⟩+ E⟨2⟩+ K2q) mod Q

= (A⟨1⟩⋅S + B⟨1⟩) ⋅(A⟨2⟩⋅S + B⟨2⟩) mod Q

= D0 + D1 ⋅S + D2 ⋅S2 mod Q ▹ D0 = B⟨1⟩B⟨2⟩, D1 = A⟨1⟩B⟨2⟩+ A⟨2⟩B⟨1⟩, D2 = A⟨1⟩A⟨2⟩

= 𝖱𝖫𝖶𝖤S,σ−1(𝖼𝗍α) + 𝖱𝖫𝖶𝖤S,σ−1(𝖼𝗍β) −∑ i=1l(Ei′⋅D2,i) mod Q

▹ 𝖼𝗍α = (D1,D0) = (Aα,Bβ), 𝖼𝗍β = ⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(D2),𝖱𝖫𝖾𝗏S,σβ,l(S2)⟩= (Aβ,Bβ)

= 𝖱𝖫𝖶𝖤S,σ−1(𝖼𝗍α + 𝖼𝗍β) −∑ i=1l(Ei′⋅D2,i) mod Q

= 𝖱𝖫𝖶𝖤S,σ−1((Aα+β,Bα+β)) −∑ i=1l(Ei′⋅D2,i) mod Q ▹ Aα+β = Aα + Aβ, Bα+β = Bα + Bβ

From the above, we extract the following relation:

(ΔM⟨1⟩+ E⟨1⟩+ K1q) ⋅(ΔM⟨2⟩+ E⟨2⟩+ K2q) mod Q

= Δ2M⟨1⟩M⟨2⟩+Δ⋅(M⟨1⟩E⟨2⟩+M⟨2⟩E⟨1⟩)+q⋅(ΔM⟨1⟩K2+ΔM⟨2⟩K1+E⟨1⟩K2+E⟨2⟩K1)+K1K2q2+E⟨1⟩E⟨2⟩modQ

= 𝖱𝖫𝖶𝖤S,σ−1((Aα+β,Bα+β)) −∑ i=1l(Ei′⋅D2,i) mod Q

We can re-write the above relation as follows:

𝖱𝖫𝖶𝖤S,σ−1((Aα+β,Bα+β)) = Aα+β ⋅S + Bα+β mod Q

= Δ2M⟨1⟩M⟨2⟩+Δ⋅(M⟨1⟩E⟨2⟩+M⟨2⟩E⟨1⟩)+q⋅(ΔM⟨1⟩K2+ΔM⟨2⟩K1+E⟨1⟩K2+E⟨2⟩K1)+K1K2q2+E⟨1⟩E⟨2⟩+∑ i=1l(Ei′⋅D2,i)modQ

To verbally interpret the above relation, decrypting the synthetically generated ciphertext (Aα+β,Bα+β) and applying a reduction modulo Q to it gives us Δ2M⟨1⟩M⟨2⟩ with a lot of noise terms. Meanwhile, as explained in the beginning of this subsection, our goal is to derive a ciphertext whose decryption is ΔM⟨1⟩M⟨2⟩, also ensuring that the decrypted ciphertext’s noise is small enough to be fully eliminated by scaling down the plaintext by Δ at the end. This goal is accomplished by the final rescaling step to be explained in the next subsection.

D-2.7.4 Rescaling

The rescaling step is equivalent to converting the ciphertext (Aα+β,Bα+β) mod Q into (⌈Aα+β Δ ⌋, ⌈Bα+β Δ ⌋) mod q, where Δ = ⌊q t⌋ ≈q t. The decryption of this rescaled ciphertext (and finally scaling down by Δ) is ΔM⟨1⟩M⟨2⟩. This is demonstrated below:

⌈Aα+β Δ ⌋ ⋅S + ⌈Bα+β Δ ⌋ mod q ▹ decryption of ciphertext (⌈Aα+β Δ ⌋, ⌈Bα+β Δ ⌋) mod q

= ⌈Aα+β Δ ⌋ ⋅S + ⌈Bα+β Δ ⌋ + K3q ▹ where K3q stands for modulo reduction by q

= ⌈Aα+β Δ ⌋ ⋅S + ⌈Bα+β Δ ⌋ + K3Q Δ ▹ since Q = Δ ⋅q

= ⌈1 Δ ⋅(Aα+β ⋅S + Bα+β + K3Q)⌋ + Er ▹ Er is a rounding error

= ⌈1 Δ ⋅(Aα+β ⋅S + Bα+β mod Q)⌋ + Er

= ⌈1 Δ ⋅(Δ2M⟨1⟩M⟨2⟩+ Δ ⋅(M⟨1⟩E⟨2⟩+ M⟨2⟩E⟨1⟩)+

q ⋅(ΔM⟨1⟩K2 + ΔM⟨2⟩K1 + E⟨1⟩K2 + E⟨2⟩K1) + K1K2q2 + E⟨1⟩E⟨2⟩+ ∑ i=1l(Ei′⋅D2,i) modQ)⌋ + Er

▹as we derived at the end of §D-2.7.3

= ⌈1 Δ ⋅(Δ2M⟨1⟩M⟨2⟩+ Δ ⋅(M⟨1⟩E⟨2⟩+ M⟨2⟩E⟨1⟩)+

q ⋅(ΔM⟨1⟩K2 + ΔM⟨2⟩K1 + E⟨1⟩K2 + E⟨2⟩K1) + K1K2q2 + E⟨1⟩E⟨2⟩+ ∑ i=1l(Ei′⋅D2,i) + K4Q)⌋ + Er

▹where K4Q stands for modulo reduction by Q

= ⌈ΔM⟨1⟩M⟨2⟩+ (M⟨1⟩E⟨2⟩+ M⟨2⟩E⟨1⟩) + q ⋅(M⟨1⟩K2 + M⟨2⟩K1)

+ 1 Δ ⋅q ⋅(E⟨1⟩K2 + E⟨2⟩K1) + 1 Δ ⋅(K1K2q2 + E⟨1⟩E⟨2⟩+ ∑ i=1l(Ei′⋅D2,i)) + K5q⌋

▹ where K5q = K4q + M⟨1⟩qK2 + M⟨2⟩K1q

= ⌈ΔM⟨1⟩M⟨2⟩+ (M⟨1⟩E⟨2⟩+ M⟨2⟩E⟨1⟩) + q ⋅(M⟨1⟩K2 + M⟨2⟩K1) + (t + 𝜖) ⋅(E⟨1⟩K2 + E⟨2⟩K1)

+ 1 Δ ⋅(K1K2q2 + E⟨1⟩E⟨2⟩+ ∑ i=1l(Ei′⋅D2,i)) + K5q⌋

▹where 𝜖 = q Δ −q q t = q ⌊q t⌋ −q q t ≈0, thus we substituted q Δ = 𝜖 + t

= ⌈ΔM⟨1⟩M⟨2⟩+ (M⟨1⟩E⟨2⟩+ M⟨2⟩E⟨1⟩) + q ⋅(M⟨1⟩K2 + M⟨2⟩K1) + (t + 𝜖) ⋅(E⟨1⟩K2 + E⟨2⟩K1)

+ K1K2q2 Δ + E⟨1⟩E⟨2⟩+ ∑ i=1l(Ei′⋅D2,i) Δ + K5q⌋

▹ Now, let 𝜖′ = K1K2q2 Δ −K1K2q2 q t = K1K2q2 ⌊q t⌋ −K1K2q2 q t ≈0.

Thus, we will substitute K1K2q2 Δ = K1K2q2 q t + 𝜖′ = K1K2qt + 𝜖′

= ΔM⟨1⟩M⟨2⟩+ (M⟨1⟩E⟨2⟩+ M⟨2⟩E⟨1⟩) + q ⋅(M⟨1⟩K2 + M⟨2⟩K1)

+ (t + 𝜖) ⋅(E⟨1⟩K2 + E⟨2⟩K1) + K1K2qt + 𝜖′+ K5q + ⌈E⟨1⟩E⟨2⟩+ ∑ i=1l(Ei′⋅D2,i) Δ ⌋

= ΔM⟨1⟩M⟨2⟩+ 𝜖″ + K6q

▹ where K6q = K5q + q ⋅(M⟨1⟩K2 + M⟨2⟩K1) + K1K2qt,

𝜖″ = M⟨1⟩E⟨2⟩+ M⟨2⟩E⟨1⟩+ (t + 𝜖) ⋅(E⟨1⟩K2 + E⟨2⟩K1) + ⌈E⟨1⟩E⟨2⟩+ ∑ i=1l(Ei′⋅D2,i) Δ ⌋

= ΔM⟨1⟩M⟨2⟩+ 𝜖″ mod q

In conclusion, the ciphertext (⌈Aα+β Δ ⌋, ⌈Bα+β Δ ⌋) mod q successfully decrypts to ΔM⟨1⟩M⟨2⟩ if 𝜖″ < Δ 2 ≈ q 2t.

Noise Analysis: Among the terms of 𝜖″, let’s analyze the noise growth of the (t + 𝜖) ⋅(E⟨1⟩K2 + E⟨2⟩K1) term after ciphertext-to-ciphertext multiplication. Each coefficient of K1 is at most n, because A⟨1⟩⋅S + B⟨1⟩ = ΔM + E⟨1⟩+ K1q, where the maximum possible coefficient value of A⟨1⟩⋅S + B⟨1⟩ is q ⋅n. And the same is true for the coefficients of K2. Therefore, after scaling down (t + 𝜖) ⋅(E⟨1⟩K2 + E⟨2⟩K1) by Δ upon the final decryption stage, this term’s down-scaled noise gets bound by:

1 Δ ⋅(t + 𝜖) ⋅(E⟨1⟩K2 + E⟨2⟩K1) ≈t q ⋅(t + 𝜖) ⋅(E⟨1⟩K2 + E⟨2⟩K1) < nt ⋅(t + 𝜖) q ⋅(E⟨1⟩+ E⟨2⟩)

This implies that for correct decryption, nt ⋅(t + 𝜖) q ⋅(E⟨1⟩+ E⟨2⟩) has to be smaller than 0.5. In other words, E⟨1⟩+ E⟨2⟩ has to be smaller than q 2nt ⋅(t + 𝜖). We can do noise analysis for all other terms for 𝜖″ in a similar manner. Importantly, upon decryption, the aggregation of all these noise terms’ down-scaled values has to be smaller than 0.5 for correct decryption.

Modulus Switch v.s. Rescaling: Notice in the rescaling process, multiplying 1 Δ to Aα+β and Bα+β results in two effects: (1) converts Δ2M⟨1⟩M⟨2⟩ into ΔM⟨1⟩M⟨2⟩; (2) switches the modulus of the mod-raised ciphertexts from Q →q. In fact, modulus switch and rescaling are closely equivalent to each other. Modulus switch is a process of changing a ciphertext’s modulus (e.g., q →q′), while preserving the property that the decryption of both ciphertexts results in the same plaintext. On the other hand, rescaling refers to the process of changing the scaling factor of a plaintext within a ciphertext (e.g., Δ →Δ′). Modulus switch inevitably changes the scaling factor of the plaintext within the target ciphertext, and rescaling also inevitably changes the modulus of the ciphertext that contains the plaintext (as shown in §D-2.7.4). Therefore, these two terms can be used interchangeably.

D-2.7.5 Summary

To put all things together, BFV’s ciphertext-to-ciphertext multiplication is summarized as follows:

⟨Summary D-2.7.5⟩ BFV’s Ciphertext-to-Ciphertext Multiplication

Suppose we have the following two RLWE ciphertexts:

𝖱𝖫𝖶𝖤S,σ(ΔM⟨1⟩) = (A⟨1⟩,B⟨1⟩) mod q, where B⟨1⟩ = −A⟨1⟩⋅S + ΔM⟨1⟩+ E⟨1⟩mod q

𝖱𝖫𝖶𝖤S,σ(ΔM⟨2⟩) = (A⟨2⟩,B⟨2⟩) mod q, where B⟨2⟩ = −A⟨2⟩⋅S + ΔM⟨2⟩+ E⟨2⟩mod q

Multiplication between these two ciphertexts is performed as follows:

1.
ModRaise

Forcibly raise the modulus of the ciphertexts (A⟨1⟩,B⟨1⟩) mod q and (A⟨2⟩,B⟨2⟩) mod q to Q (where Q = q ⋅Δ) as follows:

(A⟨1⟩,B⟨1⟩) mod Q

(A⟨2⟩,B⟨2⟩) mod Q

2.
Multiplication

Compute the following polynomial multiplications in modulo Q:

D0 = B⟨1⟩B⟨2⟩mod Q

D1 = B⟨2⟩A⟨1⟩+ B⟨1⟩A⟨2⟩mod Q

D2 = A⟨1⟩⋅A⟨2⟩mod Q

3.
Relinearization

Compute the following:

𝖼𝗍α = (D1,D0)

𝖼𝗍β = ⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(D2), 𝖱𝖫𝖾𝗏S,σβ,l(S2)⟩.

𝖼𝗍α+β = 𝖼𝗍α + 𝖼𝗍β

Then, the following property holds:

𝖱𝖫𝖶𝖤S,σ−1(𝖱𝖫𝖶𝖤S,σ(Δ2 ⋅M⟨1⟩⋅M⟨2⟩)) ≈𝖱𝖫𝖶𝖤S,σ−1(𝖼𝗍α+β)

4.
Rescaling

Update 𝖼𝗍α+β = (Aα+β,Bα+β) mod Q to 𝖼𝗍’α+β = (⌈Aα+β Δ ⌋, ⌈Bα+β Δ ⌋) mod q.

This plaintext rescaling process can be also viewed as a modulus switch of the ciphertext 𝖼𝗍α+β from Q →q.

Note that after the ciphertext-to-ciphertext multiplication, the plaintext scaling factor Δ = ⌊q t⌋, the ciphertext modulus q, and the private key S stay the same as before.

The Purpose of ModRaise: When we multiply polynomials at the second step of ciphertext-to-ciphertext multiplication (§D-2.7.2), the underlying plaintext within the ciphertext temporarily grows to Δ2M⟨1⟩M⟨2⟩, which exceeds the allowed maximum boundary q for the plaintext (Summary D-2.3 in §D-2.3). After this point, applying modulo-q reduction to the intermediate result will irrevocably corrupt the plaintext. To avoid the corruption of the plaintext when it grows to Δ2M⟨1⟩M⟨2⟩, we temporarily increase the ciphertext modulus from q →Q, which is sufficiently large to hold Δ2M⟨1⟩M⟨2⟩ without wrapping around the boundary of the ciphertext modulus.

Swapping the Order of Relinearization and Rescaling: The order of relinearization and rescaling is interchangeable. Running rescaling before relinearization reduces the size of the ciphertext modulus, and therefore the subsequent relinearization can be executed faster.

D-2.7.6 Application to TFHE’s Ciphertext-to-Ciphertext Multiplication

In §D-1.6, we learned how TFHE performs ciphertext-to-ciphertext multiplication between LWE and GSW ciphertexts. However, this method is computationally expensive because it requires circuit bootstrapping to convert an LWE ciphertext into a GSW ciphertext. Such an overhead can be avoided if we directly apply BFV’s ciphertext-to-ciphertext multiplication strategy to TFHE’s LWE ciphertexts. Given two LWE ciphertexts to multiply, they hold the following relations:

𝖫𝖶𝖤s→,σ(Δm⟨1⟩+ e⟨1⟩) = (a→⟨1⟩,b⟨1⟩) ⇒a→⟨1⟩⋅s→ + b⟨1⟩ = Δm⟨1⟩+ e⟨1⟩+ k1q

𝖫𝖶𝖤s→,σ(Δm⟨2⟩+ e⟨2⟩) = (a→⟨2⟩,b⟨2⟩) ⇒a→⟨2⟩⋅s→ + b⟨2⟩ = Δm⟨2⟩+ e⟨2⟩+ k2q

Multiplying these two equations yields the following relation:

(Δm⟨1⟩+ e⟨1⟩+ k1q) ⋅(Δm⟨2⟩+ e⟨2⟩+ k2q) ▹≈Δ2m⟨1⟩m⟨2⟩mod q

= (a→⟨1⟩⋅s→ + b⟨1⟩) ⋅(a→⟨2⟩⋅s→ + b⟨2⟩)

= b⟨1⟩b⟨2⟩+ (b⟨2⟩a→⟨1⟩+ b⟨1⟩a→⟨2⟩) ⋅s→ + (a→⟨1⟩⋅s→) ⋅(a→⟨2⟩⋅s→)

= b⟨1⟩b⟨2⟩ d0 + (b⟨2⟩a→⟨1⟩+ b⟨1⟩a→⟨2⟩) d1 ⋅s→ + (a→⟨1⟩⊗a→⟨2⟩) d→2 ⋅(s→ ⊗s→)

= d0 + d1 ⋅s→ + d→2 ⋅s→⊗ ▹ where s→⊗ = s→ ⊗s→

, where ⊗ denotes an outer product of two vectors. For example, given two n-length vectors v→ and u→, v→ ⊗u→ is equivalent to an n2-length vector that concatenates the following n distinct n-length vectors: v0 ⋅u→,v1 ⋅u→,⋯vn−1 ⋅u→. Notice that the above relation is similar to that we derived in BFV’s ciphertext-to-ciphertext multiplication. Therefore, similar to the remaining steps in BFV, we can relinearize and rescale this LWE term as follows:

⟨Summary D-2.7.5⟩ TFHE’s Ciphertext-to-Ciphertext Multiplication - Method 2

Suppose we have the following two LWE ciphertexts:

𝖫𝖶𝖤s→,σ(ΔM⟨1⟩) = (a→⟨1⟩,b⟨1⟩) mod q, where b⟨1⟩ = −a→⟨1⟩⋅s→ + Δm⟨1⟩+ e⟨1⟩mod q

𝖫𝖶𝖤s→,σ(ΔM⟨2⟩) = (a→⟨2⟩,b⟨2⟩) mod q, where b⟨2⟩ = −a→⟨2⟩⋅s→ + Δm⟨2⟩+ e⟨2⟩mod q

Multiplication between these two LWE ciphertexts is performed as follows:

1.
ModRaise

Forcibly raise the modulus of the ciphertexts (a→⟨1⟩,b⟨1⟩) mod q and (a→⟨2⟩,b⟨2⟩) mod q to Q (where Q = q ⋅Δ) as follows:

(a→⟨1⟩,b⟨1⟩) mod Q

(a→⟨2⟩,b⟨2⟩) mod Q

2.
Multiplication

Compute the following polynomial multiplications in modulo Q:

d0 = b⟨1⟩b⟨2⟩mod Q

d1 = b⟨2⟩a→⟨1⟩+ b⟨1⟩a→⟨2⟩mod Q

d→2 = a→⟨1⟩⊗a→⟨2⟩mod Q

3.
Relinearization

Compute the following:

𝖼𝗍α = (d1,d0)

𝖼𝗍β = ⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(d→2), 𝖫𝖾𝗏s→,σβ,l(s→⊗)⟩.

𝖼𝗍α+β = 𝖼𝗍α + 𝖼𝗍β

Then, the following property holds:

𝖫𝖶𝖤s→,σ−1(𝖫𝖶𝖤s→,σ(Δ2 ⋅m⟨1⟩⋅m⟨2⟩)) ≈𝖫𝖶𝖤s→,σ−1(𝖼𝗍α+β)

4.
Rescaling

Update 𝖼𝗍α+β = (a→α+β,bα+β) mod Q to 𝖼𝗍’α+β = (⌈a→α+β Δ ⌋, ⌈bα+β Δ ⌋) mod q.

This plaintext rescaling process can be also viewed as a modulus switch of the ciphertext 𝖼𝗍α+β from Q →q.

Note that after the ciphertext-to-ciphertext multiplication, the plaintext scaling factor Δ = q t, the ciphertext modulus q, and the private key s→ stay the same as before.
