C-5 GLWE Key Switching

- Reference: TFHE Deep Dive - Part III - Key switching and leveled multiplications [9]

Key switching is a process to change a GLWE ciphertext’s secret key from S to a new key S′ without decrypting the ciphertext. This is equivalent to converting a ciphertext 𝖦𝖫𝖶𝖤S,σ(ΔM + E) into a new ciphertext 𝖦𝖫𝖶𝖤S′,σ(ΔM + E −E′).

Remember that 𝖦𝖫𝖶𝖤S,σ(ΔM + E) = (A0,A1,…,Ak−1,B) ∈R⟨n,q⟩k+1

, where B = ∑ i=0k−1(Ai ⋅Si) + Δ ⋅M + E

, and the secret key S is a list of k polynomials: S = (S0,S1,…,Sk−1)

Also, remember that GLev (§B-5) is defined as follows:

𝖦𝖫𝖾𝗏S,σβ,l(M) ={GLWES,σ( q βi ⋅M + Ei)}i=1l ∈R⟨n,q⟩(k+1)⋅l

Now, let us denote each of the k key-switching keys as follows:

𝐾𝑆𝐾i = 𝖦𝖫𝖾𝗏S′,σβ,l(Si)
= (𝖦𝖫𝖶𝖤S′,σ ( q β1Si + Ei,1) ,𝖦𝖫𝖶𝖤S′,σ ( q β2Si + Ei,2) ,…,𝖦𝖫𝖶𝖤S′,σ ( q βlSi + Ei,l)) ∈R⟨n,q⟩(k′+1)⋅l

, which is a list of GLWE encryptions of the secret key S by S′, and the new secret key S′ is a list of k′ polynomials, (S0′,S1′,…,Sk′−1′). Then, the GLWE ciphertext’s key can be switched from S →S′ as follows:

⟨Summary C-5⟩ GLWE Key Switching

Given 𝖦𝖫𝖶𝖤S,σ(ΔM + E) = (A,B),

𝖦𝖫𝖶𝖤S′,σ(ΔM + E′) = (0,0,…,0k′ ,B) −∑ i=0k−1Ai ⋅Si

= (0,0,…,0,B) −∑ i=0k−1⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai),𝐾𝑆𝐾i⟩

Proof.

1.
Given the principle of polynomial decomposition (§A-6.2) and the polynomial Ai ∈ ℤq[x]∕(xn + 1), its decomposition is as follows:

𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai) = (Ai,1,Ai,2,…,Ai,l), where

Ai = Ai,1 q β1 + Ai,2 q β2 + ⋯ + Ai,l q βl

2.
⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai),𝐾𝑆𝐾i⟩
= Ai,1 ⋅𝖦𝖫𝖶𝖤S′,σ ( q β1Si + Ei,1) + Ai,2 ⋅𝖦𝖫𝖶𝖤S′,σ ( q β2Si + Ei,2) + ⋯ + Ai,l ⋅𝖦𝖫𝖶𝖤S′,σ ( q βlSi + Ei,l)

▹ where each GLWE ciphertext is an encryption of Si q β,Si q β2,⋯,Si q βl as plaintext with the plaintext scaling factor 1

= 𝖦𝖫𝖶𝖤S′,σ ((Ai,1 q β1 + Ai,2 q β2 + ⋯ + Ai,l q βl) ⋅Si + Ei) ▹ where Ei = ∑ j=1lAi,jEi,j

= 𝖦𝖫𝖶𝖤S′,σ ((Ai) ⋅Si + Ei) = 𝖦𝖫𝖶𝖤S′,σ(AiSi + Ei)

3.
∑ i=0k−1⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai),𝐾𝑆𝐾i⟩= ∑ i=0k−1(𝖦𝖫𝖶𝖤S′,σ(AiSi + Ei))

= 𝖦𝖫𝖶𝖤S′,σ (∑ i=0k−1AiSi + Ei)

4.
B is equivalent to a trivial GLWE encryption with S′, where its every A0,A1,…,Ak′−1 is 0. That is, 𝖦𝖫𝖶𝖤S′,σ(B) = (0,0,…k′ ,B). Note that 𝖦𝖫𝖶𝖤S′,σ(B) can be created without the knowledge of S′, because its all AiSi terms are 0.

5.
𝖦𝖫𝖶𝖤S′,σ(B) −𝖦𝖫𝖶𝖤S′,σ (∑ i=0k−1(AiSi + Ei))

= 𝖦𝖫𝖶𝖤S′,σ(B −∑ i=0k−1(AiSi + Ei))

= 𝖦𝖫𝖶𝖤S′,σ(ΔM + E′) ▹ where E′ = E −∑ i=0k−1Ei

6.
If we explicitly expand the above relation,

𝖦𝖫𝖶𝖤S′,σ(B) −𝖦𝖫𝖶𝖤S′,σ (∑ i=0k−1(AiSi + Ei))

= (0,0,…k′ ,B) −(A0′,A 1′,…,A k′−1′k′ ,B′) ▹ where B′ = ∑ i=0k′−1 Ai′Si′+ ∑ i=0k−1AiSi + ∑ i=0k−1Ei

(−A0′,−A 1′,…,−A k′−1′k′ , B −B′)

The decryption of the above ciphertext gives us:

𝖦𝖫𝖶𝖤S′,σ−1( (−A0′,−A 1′,…,−A k′−1′k′ , B −B′) )

= B −B′−∑ i=0k′−1 −Ai′Si′

= B −∑ i=0k′−1 Ai′Si′−∑ i=0k−1AiSi −∑ i=0k−1Ei + ∑ i=0k′−1 Ai′Si′

= B −∑ i=0k−1AiSi −∑ i=0k−1Ei

= ΔM + E −∑ i=0k−1Ei

= ΔM + E′ ≈ΔM + E ▹ where E′ = E −∑ i=0k−1Ei

Strictly speaking, B −∑ i=0k−1AiSi = ΔM + E + Kq where K is a polynomial representing the wrap-around coefficient values as multiples of q. However, since all the above computations are done in modulo q, the Kq term gets eliminated.

7.
Thus, (0,0,…,B) −∑ i=0k−1⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai),𝐾𝑆𝐾i⟩= 𝖦𝖫𝖶𝖤S′,σ(ΔM + E′) ≈𝖦𝖫𝖶𝖤S′,σ(ΔM + E)

𝖦𝖫𝖶𝖤S′,σ(ΔM + E′) is an encryption of plaintext ΔM with the plaintext scaling factor 1. However, we can re-interpret this ciphertext as an encryption of plaintext M with the plaintext scaling factor Δ. This way, we can recover the ciphertext’s original scaling factor Δ without any additional computation.

□
