C-5 GLWE Key Switching

- Reference: TFHE Deep Dive - Part III - Key switching and leveled multiplications [9]

Key switching is a process to change a GLWE ciphertext’s secret key from S to a new key S′ without decrypting the ciphertext. This is equivalent to converting a ciphertext 𝖦𝖫𝖶𝖤S,σ(ΔM) into a new ciphertext 𝖦𝖫𝖶𝖤S′,σ(ΔM).

Remember that 𝖦𝖫𝖶𝖤S,σ(ΔM) = (A0,A1, ⋯ ,Ak−1,B) ∈R⟨n,q⟩k+1

, where B = ∑ ⁡ i=0k−1(Ai ⋅Si) + Δ ⋅M + E

, and the secret key S is a list of k polynomials: S = (S0,S1, ⋯ Sk−1)

Also, remember that GLev (§B-5) is defined as follows:

𝖦𝖫𝖾𝗏S,σβ,l(M) ={GLWES,σ( q βi ⋅M)}i=1l ∈R⟨n,q⟩(k+1)⋅l

Now, let’s denote each of the k key-switching keys as follows:

𝐾𝑆𝐾i = 𝖦𝖫𝖾𝗏S′,σβ,l(Si)
= (𝖦𝖫𝖶𝖤S′,σ ( q β1Si) ,𝖦𝖫𝖶𝖤S′,σ ( q β2Si) , ⋯ ,𝖦𝖫𝖶𝖤S′,σ ( q βlSi)) ∈R⟨n,q⟩(k+1)⋅l

, which is a list of GLWE encryptions of the secret key S by S′. Then, the GLWE ciphertext’s key can be switched from S →S′ as follows:

⟨Summary C-5⟩ GLWE Key Switching

𝖦𝖫𝖶𝖤S′,σ(ΔM) = (0,0, ⋯ 0︷k,B) −∑ ⁡ i=0k−1Ai ⋅Si

= (0,0, ⋯ 0,B) −∑ ⁡ i=0k−1⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai),𝐾𝑆𝐾i⟩

Proof.

1.
Given the principle of polynomial decomposition (§A-6.2) and the polynomial Ai ∈ ℤq[x]∕(xn + 1), its decomposition is as follows:

𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai) = (A⟨i,1⟩,A⟨i,2⟩, ⋯ A⟨i,l⟩), where

Ai = A⟨i,1⟩ q β1 + A⟨i,2⟩ q β2 + ⋯ + A⟨i,l⟩ q βl

2.
⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai),𝐾𝑆𝐾i⟩
= A⟨i,1⟩⋅𝖦𝖫𝖶𝖤S′,σ ( q β1Si) + A⟨i,2⟩⋅𝖦𝖫𝖶𝖤S′,σ ( q β2Si) +  ⋯  + A⟨i,l⟩⋅𝖦𝖫𝖶𝖤S′,σ ( q βlSi) # where each GLWE ciphertext is an encryption of Si q β,Si q β2,⋯,Si q βl as plaintext with the plaintext scaling factor 1

= 𝖦𝖫𝖶𝖤S′,σ ((A⟨i,1⟩ q β1 + A⟨i,2⟩ q β2 +  ⋯ A⟨i,l⟩ q βl) ⋅Si)

= 𝖦𝖫𝖶𝖤S′,σ ((Ai) ⋅Si) = 𝖦𝖫𝖶𝖤S′,σ(AiSi)

3.
∑ ⁡ i=0k−1⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai),𝐾𝑆𝐾i⟩= ∑ i=0k−1(𝖦𝖫𝖶𝖤S′,σ(AiSi))

= 𝖦𝖫𝖶𝖤S′,σ (∑ ⁡ i=0k−1AiSi)

4.
B is equivalent to a trivial GLWE encryption with S′, where its every A1,A2,⋯ is 0. That is, 𝖦𝖫𝖶𝖤S′,σ(B) = (0,0, ⋯ ︷k,B). Note that 𝖦𝖫𝖶𝖤S′,σ(B) can be created without the knowledge of S′, because its all AiSi terms are 0.

5.
𝖦𝖫𝖶𝖤S′,σ(B) −𝖦𝖫𝖶𝖤S′,σ (∑ ⁡ i=0k−1(AiSi)) = 𝖦𝖫𝖶𝖤S′,σ(B −∑ ⁡ i=0k−1(AiSi)) = 𝖦𝖫𝖶𝖤S′,σ(ΔM + E)

6.
To expand the above relation,

𝖦𝖫𝖶𝖤S′,σ(B) −𝖦𝖫𝖶𝖤S′,σ (∑ ⁡ i=0k−1(AiSi))

= (0,0, ⋯ ︷k,B) −(A0′,A 1′, ⋯ ,A k′︷k,B′) # where B′ = ∑ ⁡ i=0k−1Ai′Si′+ ∑ i=0k−1AiSi + E′

(−A0′,−A 1′, ⋯ ,−A k′︷k, B −B′)

The decryption of the above ciphertext gives us:

𝖦𝖫𝖶𝖤S′,σ−1( (−A0′,−A 1′, ⋯ ,−A k′︷k, B −B′) )

= B −B′−∑ ⁡ i=0k−1 −Ai′Si′

= B −∑ ⁡ i=0k−1Ai′Si′−∑ i=0k−1AiSi −E′+ ∑ i=0k−1Ai′Si′

= B −∑ ⁡ i=0k−1AiSi −E′

= ΔM + E −E′ ≈ΔM + E

Strictly speaking, B −∑ ⁡ i=0k−1AiSi = ΔM + E + 𝐾𝑞 where K is a polynomial to represent the wrap-around coefficient values as multiples of q. However, since all the above computations are done in modulo q, the 𝐾𝑞 term gets eliminated.

7.
Thus, (0,0, ⋯ ,B) −∑ ⁡ i=0k−1⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(Ai),𝐾𝑆𝐾i⟩≈𝖦𝖫𝖶𝖤S′,σ(ΔM)

𝖦𝖫𝖶𝖤S′,σ(ΔM) is an encryption of plaintext ΔM with the plaintext scaling factor 1. However, we can theoretically see this as an encryption of plaintext M with the plaintext scaling factor Δ. This way, we can recover the ciphertext’s original scaling factor Δ without any additional computation.

□
