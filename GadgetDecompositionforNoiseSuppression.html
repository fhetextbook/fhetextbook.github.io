C-3.1 Gadget Decomposition for Noise Suppression

In ciphertext-to-plaintext multiplication Λ ⋅𝖦𝖫𝖶𝖤S,σ(M), the noise E grows to E′ = Λ ⋅E. To limit this noise growth, we introduce a technique based on decomposing Λ (§A-6.1) and a GLev encryption (§B-5.1) of M as follows:

Λ = Λ1 q β1 + Λ2 q β2 + ⋯ + Λl q βl→𝖣𝖾𝖼𝗈𝗆𝗉β,l(Λ) = (Λ1,Λ2,⋯,Λl)

𝖦𝖫𝖾𝗏S,σβ,l(ΔM) = {𝖦𝖫𝖶𝖤S,σ (ΔM q β1) ,𝖦𝖫𝖶𝖤S,σ (ΔM q β2) ,⋯𝖦𝖫𝖶𝖤S,σ (ΔM q βl) }

We will encrypt the plaintext M as 𝖦𝖫𝖾𝗏S,σβ,l(ΔM) instead of 𝖦𝖫𝖶𝖤S,σ(ΔM), and compute 𝖣𝖾𝖼𝗈𝗆𝗉(𝖣𝖾𝖼𝗈𝗆𝗉β,l(Λ) ⋅𝖦𝖫𝖾𝗏S,σβ,l(ΔM) instead of Λ ⋅𝖦𝖫𝖶𝖤S,σ(ΔM). Notice that the results of both computations are the same as follows:

𝖣𝖾𝖼𝗈𝗆𝗉β,l(Λ) ⋅𝖦𝖫𝖾𝗏S,σβ,l(ΔM)

= (Λ1,Λ2,⋯,Λl) ⋅ (𝖦𝖫𝖶𝖤S,σ ( q βΔM ), 𝖦𝖫𝖶𝖤S,σ ( q β2ΔM ), ⋯, 𝖦𝖫𝖶𝖤S,σ ( q βlΔM ))

= Λ1 ⋅𝖦𝖫𝖶𝖤S,σ ( q βΔM ) + Λ2 ⋅𝖦𝖫𝖶𝖤S,σ ( q β2ΔM ) + ⋯ + Λl ⋅𝖦𝖫𝖶𝖤S,σ ( q βlΔM )

= 𝖦𝖫𝖶𝖤S,σ (Λ1 ⋅q βM ) + 𝖦𝖫𝖶𝖤S,σ (Λ2 ⋅ q β2M ) + ⋯ + 𝖦𝖫𝖶𝖤S,σ (Λl ⋅ q βlM )

= 𝖦𝖫𝖶𝖤S,σ (Λ1 ⋅q βΔM + Λ2 ⋅ q β2ΔM + ⋯ + Λl ⋅ q βlΔM )

= 𝖦𝖫𝖶𝖤S,σ ((Λ1 ⋅q β + Λ2 ⋅ q β2 + ⋯ + Λl ⋅ q βl) ⋅ΔM )

= 𝖦𝖫𝖶𝖤S,σ (Λ ⋅ΔM )

= Λ ⋅𝖦𝖫𝖶𝖤S,σ(ΔM)

While the computation results are the same, as we decompose Λ into smaller plaintext polynomials Λ1,Λ2,⋯,Λl, the generated noise by each of l plaintext-to-ciphertext multiplications becomes smaller. Given the noise of each GLWE ciphertext in the GLev ciphertext is Ei, the final noise of the ciphertext-to-plaintext multiplication is ∑ ⁡ i=1lΛi ⋅Ei, which is much smaller than Λ ⋅E (because the coefficients of each decomposed polynomial Λi are significantly smaller than those of Λ). This is visually depicted in Figure 11.

However, we cannot use this decomposition technique to the resulting ciphertext again, because the output of this algorithm is a GLWE ciphertext and converting it into a GLev ciphertext without decrypting it costs much noise and computation time (as we need to multiply the GLWE ciphertext by q β, q β2,⋯, q βl).

As a more efficient technique to re-initialize the noise E, we will describe TFHE’s noise bootstrapping technique in §D-1.8.
