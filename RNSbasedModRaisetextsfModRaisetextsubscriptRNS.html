D-5.3 RNS-based ModRaise: ModRaiseRNS

- Reference: A Full RNS Variant of Approximate Homomorphic Encryption [26]

ModRaise is an operation that raises a ciphertext’s modulus from q to qb (where q ≪qb). We used ModRaise in BFV’s ciphertext-to-ciphertext multiplication (Summary D-2.7.5 in §D-2.7.5) and in CKKS’s modulus bootstrapping (Summary D-3.13.7 in §D-3.13.7). The RNS-based ModRaise operation is designed as follows:

⟨Summary D-5.3⟩ ModRaiseRNS

Input: (x1,x2,⋯,xk) ∈ ℤq1 × ℤq2 ×⋯ × ℤqk ▹ which represents the big value x ∈ ℤq

𝖬𝗈𝖽𝖱𝖺𝗂𝗌𝖾𝖱𝖭𝖲({xi}i=1k,q,qb) ▹ where q and b are co-prime

= 𝖥𝖺𝗌𝗍𝖡𝖢𝗈𝗇𝗏𝖱𝖭𝖲({xi}i=1k,q,qb)

= (x1,x2,⋯,xk,𝖥𝖺𝗌𝗍𝖡𝖢𝗈𝗇𝗏({xi}i=1k,q,b))

= (x1,x2,⋯,xk,c1,c2,⋯,cl) ∈ ℤq1 × ℤq2 ×⋯ × ℤqk × ℤb1 × ℤb2 ×⋯ × ℤbl

= (χ1,χ2,⋯,χk+l) ∈ ℤq1 × ℤq2 ×⋯ × ℤqk × ℤb1 × ℤb2 ×⋯ × ℤbl

▹ which represents the value χ ∈ ℤqb

The relationship between χ and x is as follows:

χ ≡x + u ⋅q mod qb ▹ the noise generated by ModRaiseRNS is |uq|qb (where integer |u|≤k 2 + 1)

χ ≡x mod q

Proof.

In §D-5.1, we proved that x′ = ∑ i=1k|xi ⋅zi|qi ⋅yi = x + u ⋅q (where integer |u|≤k 2 + 1). Therefore, the following holds:

x′ ≡xi mod qi for i ∈[1,k] ▹ since x′ = x + u ⋅q ≡xi mod qi (as qi divides q, thus u ⋅q ≡0(modqi), and x ≡xi mod qi)

x′ ≡cj mod bj for j ∈[1,l] ▹ where each cj = x + uq mod bj

Therefore, x′mod qb can be represented as the following RNS residues:

(x1,x2,⋯,xk,c1,c2,⋯,cl) ∈ ℤq1 × ℤq2 ×⋯ × ℤqk × ℤb1 × ℤb2 ×⋯ × ℤbl

= (x1,x2,⋯,xk,𝖥𝖺𝗌𝗍𝖡𝖢𝗈𝗇𝗏({xi}i=1k,q,b)) ∈ ℤq1 × ℤq2 ×⋯ × ℤqk × ℤb1 × ℤb2 ×⋯ × ℤbl

Our ideal goal of mod-raising x ∈ ℤq from q →qb is to derive an RNS vector of x mod qb. However, the above RNS vector represents x′mod qb, where x′ = x + uq (with integer |u|≤k 2 + 1). Therefore, we can interpret the above RNS vector as representing x mod qb with the additional noise |uq|qb. □
